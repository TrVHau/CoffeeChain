> **[Unit-3] BE-Member-3** — Maintained by BE-Member-3

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Docker Desktop | Latest | https://docs.docker.com/get-docker/ |
| Docker Compose | v2+ | bundled with Docker Desktop |
| Fabric binaries | 2.5 | See below |
| fabric-ca-client | 1.5 | See below |
| Java | 21 | For chaincode build |
| Gradle (wrapper) | 8.x | `./gradlew` inside `chaincode/` |

### Install Fabric binaries

```bash
curl -sSL https://bit.ly/2ysbOFE | bash -s -- 2.5.0 1.5.0
# This installs: cryptogen, configtxgen, peer, fabric-ca-client
export PATH="$PWD/bin:$PATH"
```

## Directory Structure

```
network/
├── configtx.yaml           ← channel + org definitions (for configtxgen)
├── crypto-config.yaml      ← identity material spec (for cryptogen)
├── docker-compose.yaml     ← all containers
├── scripts/
│   ├── setup-network.sh    ← Step 1: crypto + genesis + channel + peers join
│   ├── deploy-chaincode.sh ← Step 2: build JAR, install, approve, commit
│   └── register-users.sh   ← Step 3: CA register + enroll 5 demo users
└── channel-artifacts/      ← generated by setup-network.sh (git-ignored)
└── crypto-config/          ← generated by setup-network.sh (git-ignored)
```

## Quick Start (Demo)

```bash
# 0. Clone repo and cd into network/
cd network/
chmod +x scripts/*.sh

# 1. Bring up Fabric network (orderer, peers, CAs)
./scripts/setup-network.sh

# 2. Deploy chaincode
./scripts/deploy-chaincode.sh

# 3. Register demo users
./scripts/register-users.sh

# 4. Start remaining services (PostgreSQL, IPFS, backend, frontend)
docker compose up -d postgres ipfs backend frontend
```

## Detailed Steps

### Step 1: setup-network.sh

Executes:
1. `cryptogen generate` — creates `crypto-config/` with all certs and keys
2. `configtxgen` — creates `channel-artifacts/genesis.block`, `channel.tx`, anchor peer updates
3. `docker compose up` — starts orderer, peers, CAs, CouchDBs
4. `peer channel create` — creates the `coffee-traceability-channel`
5. Both peers join the channel
6. Anchor peers updated for both orgs (enables service discovery for SBE)

### Step 2: deploy-chaincode.sh

Executes:
1. `./gradlew shadowJar` — builds fat JAR (`chaincode/build/libs/chaincode.jar`)
2. `peer lifecycle chaincode package` — packages into `CoffeeTraceChaincode.tar.gz`
3. Installs on `peer0.org1` and `peer0.org2`
4. Approves for Org1MSP and Org2MSP
5. Commits chaincode to channel with **`--sequence 1`**

> For upgrades: bump `CHAINCODE_VERSION` and `CHAINCODE_SEQUENCE` and re-run the script.

### Step 3: register-users.sh

Registers and enrolls 5 users via Fabric CA:

| User | Org | Role | MSP |
|------|-----|------|-----|
| `farmer_alice` | Org1 | FARMER | Org1MSP |
| `processor_bob` | Org1 | PROCESSOR | Org1MSP |
| `roaster_charlie` | Org1 | ROASTER | Org1MSP |
| `packager_dave` | Org2 | PACKAGER | Org2MSP |
| `retailer_eve` | Org2 | RETAILER | Org2MSP |

MSP folders are created at:
```
crypto-config/peerOrganizations/<org>.example.com/users/<userId>/msp/
```

The backend's `FabricGatewayService` loads these identities at startup using `fabric.org1.usersBasePath` and `fabric.org2.usersBasePath` from `application.yaml`.

```bash
docker compose down -v

# Remove generated crypto material and artifacts
rm -rf crypto-config channel-artifacts
```

## Common Issues

| Problem | Fix |
|---------|-----|
| `cryptogen: command not found` | Add Fabric `bin/` to PATH |
| `Error: failed to create deliver client` | Orderer not ready yet — wait and retry |
| `ENDORSEMENT_POLICY_FAILURE` on acceptTransfer | Anchor peers not updated — re-run step 6 of setup-network.sh |
| PostgreSQL health check fails | Increase `sleep` after `docker compose up`, or retry |
| IPFS CID empty | IPFS container not fully started — wait ~10s after `docker compose up` |
| Backend can't load user identity | Run `register-users.sh` and check `fabric.org1.usersBasePath` in application.yaml |
